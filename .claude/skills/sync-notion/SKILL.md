# sync-notion

git logを読んでNotionロードマップの `[ ]` を `[x]` に更新する。軽量・シンプルに。

## Notionページ

`https://www.notion.so/293d9703217a8067accef9cff447b4ad`

## 手順

### 1. git logを取得（直近24時間）

```bash
git log --oneline --since="24 hours ago"
```

コミットがない場合は「since 3 days ago」に広げる。

### 2. Notionの当月セクションだけ読む

`notion-fetch` で上記URLを取得。全ページ読むが、**当月（例: ## 2026年4月）のセクションだけ**を処理対象にする。

### 3. 照合・更新

- コミットメッセージとNotionの `[ ] タスク名` をざっくり照合
- 明らかに対応するものだけ更新。迷ったらスキップ（誤更新より未更新の方がまし）
- `notion-update-page` の `update_content` で `old_str: "- [ ] タスク名"` → `new_str: "- [x] タスク名"` に変更

### 4. 結果を箇条書きで報告

更新したもの・スキップしたものを一覧で出す。確認を求める必要はない。

## 注意

- 判断が難しいものは無視する（誤ってチェックを入れない）
- 1回の呼び出しで更新は最大5件まで（それ以上あれば重要なものを選ぶ）
- ロードマップ以外のNotionページは触らない
